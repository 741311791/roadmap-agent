### 3. **Shader 背景动效不明显** 🎨 [设计特性]

**问题描述**:
- 动态 Shader 背景在截图中几乎看不出效果
- 背景看起来像静态的灰白色

**技术分析**:
Shader 效果实际上是正常工作的，但设计非常微妙：

```glsl
// 颜色定义
vec3 warmCream = vec3(0.965, 0.945, 0.918);  // #f6f1ea
vec3 sageLight = vec3(0.847, 0.878, 0.847);  // #d8e0d8
vec3 sageMid = vec3(0.627, 0.706, 0.627);    // #a0b4a0
vec3 sageDark = vec3(0.376, 0.459, 0.376);   // #607560

// 非常微妙的光晕系数
col += 0.0006 / d * (sageMid * 0.5 + sageLight * 0.5);  // L102
col += 0.001 * b / length(...) * sageDark * 0.8;         // L106

// 最终混合 20% 奶油色
col = mix(col, warmCream, 0.2);  // L121
```

**设计意图**:
- 这是**有意为之的杂志风格设计**
- 追求"优雅而不喧宾夺主"的背景效果
- 动态效果在实际浏览时更明显（流动、呼吸效果）

**可选增强方案**（如需要更明显的效果）:
```glsl
// 增强光晕系数
col += 0.002 / d * (sageMid * 0.7 + sageLight * 0.3);

// 减少奶油色混合
col = mix(col, warmCream, 0.1);

// 增加对比度
col = pow(col, vec3(0.90));  // 从 0.96 降低到 0.90
```

## 🎨 设计哲学说明

### 为什么 Shader 效果这么微妙？

**1. 杂志风格定位**
- Fast Learning 采用"奶油色纸质杂志"设计语言
- 背景应该像高端杂志的纸张 - 有质感但不抢戏
- Shader 提供"流动的纸张纹理"而非"炫酷的特效"

**2. 内容优先原则**
- 标题、文字、CTA 按钮才是主角
- 背景动效是"氛围营造"而非"视觉焦点"
- 用户应该专注于内容而非被动效分散注意力

**3. 性能与体验平衡**
- 过强的 Shader 效果会：
  - 增加 GPU 负担
  - 影响文字可读性
  - 降低页面加载性能
  
**4. 专业感与信任感**
- 教育产品需要传达专业、可靠的形象
- 过于花哨的动效可能降低信任度
- 微妙的动效体现品质而非浮夸
